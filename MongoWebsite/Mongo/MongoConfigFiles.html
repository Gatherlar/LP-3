Running Moongod & Config files

File running

To automate this a little further we created a little batch file so it easier to start (as you don’t have to keep doing the CD command into program files). Also, separating out a config file meant that it was easier to understand the parameters and set them out more tidily.

The contexts of the batch file was this...

cd "C:\Program Files\MongoDB 2.6 Standard\bin\" mongod -f C:\GitHub\Survivor\SL\DeployFiles\Config\mongodb.conf pause

The -f parameter allows you to supply a Config file, and the pause allows you to see what happens if an error occurs (otherwise the cmd window disappears!)

Config file

The config file allows lots of other things but to keep things simple I simply put the path of the database in using the ‘dbpath’ parameter

dbpath = C:\GitHub\Survivor\SL\DeployFiles\DB

Then I ensure I understood which port I was using (you can change this if you wish but the default is 27017) using the ‘port’

port = 27017

Mongo writes lots of text to this and also creates a new text file every time the service is started. So the next parameter is the ‘logpath’ which gives us control of where the log is written to. Combine this with ‘logappend’ which means there is only file and it just continues to be written to.

logpath = C:\GitHub\Survivor\SL\DeployFiles\DB\mongo.log logappend = true
