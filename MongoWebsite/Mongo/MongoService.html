Starting and Stopping the Mongo Service In Code

Introduction

So, the first thing that was quite annoying was the fact that if you close this command prompt window your mongod service will stop running.

I found that running this in a local application and having a command prompt popping up would be a little annoying.

Also, when you start and stop your application if you were using a batch file you would also have multiple instances of the ‘mongod’ service running.

In this section we discuss how to run ‘mongod’ as a service, and also demonstrate an implementation.

More Batch Files

So, I created a Batchfile which has the following in it

cd "C:\Program Files\MongoDB 2.6 Standard\bin\"
mongod -f "C:\GitHub\Survivor\SL\DeployFiles\Config\mongodb.conf" -install
-serviceName mdb27017
-serviceDisplayName "MongoDB Server Instance 27017"
-serviceDescription "MongoDB Server Instance running on 27017"
pause

It basically executes the config file and installs it as a service.
Then from the code I wrote the following to execute the script

System.Diagnostics.Process.Start(“InstallMongo.bat);

Similarly I wrote a start & stop batch file and executed that the same way. The code was pretty straight forward.

net start mdb27017 / net stop mdb27017
